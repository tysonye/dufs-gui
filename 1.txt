根据搜索结果，ngrok.exe 启动后闪退可能有以下几个常见原因：

未配置 authtoken - 需要先添加您的 authtoken：

ngrok config add-authtoken <您的TOKEN>
您可以在 ngrok dashboard 获取 authtoken。

配置文件错误 - 检查 ngrok 配置文件是否有语法错误。可以使用 ngrok config edit 查看配置。

版本过旧 - 如果 ngrok 版本太旧，可能无法连接。运行 ngrok update 更新到最新版本。

查看日志排查 - 在配置文件中添加日志输出来诊断问题：

log: C:\ngrok\ngrok.log
log_level: debug
运行诊断命令 - 使用 ngrok diagnose 命令来检测连接问题。

建议在命令行（CMD 或 PowerShell）中运行 ngrok，而不是双击运行，这样可以看到具体的错误信息。

更多信息请参考 ngrok Agent 文档 和 错误参考。

获取公网 URL 的方法：

使用 ngrok 本地 API - ngrok 启动后会在 http://127.0.0.1:4040/api/tunnels 提供 API，您可以从这里获取 URL：

curl http://127.0.0.1:4040/api/tunnels
使用 --log 参数 - 将日志输出到文件，然后解析 URL：

ngrok http 5001 --log=stdout --log-format=json
使用 ngrok API - 通过 ngrok 云端 API 查询 endpoints：

ngrok api endpoints list
如果脚本卡住，可能是因为 ngrok 在后台运行时没有将 URL 输出到 stdout。建议通过轮询本地 API (http://127.0.0.1:4040/api/tunnels) 来获取公网 URL。
